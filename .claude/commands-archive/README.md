# Archived Commands

These slash commands were moved from `.claude/commands/` to reduce cognitive load.
They still work if copied back to `commands/` when needed.

## Why archived?

Per best practices (Shrivu Shankar): "If you have a long list of complex custom slash commands,
you've created an anti-pattern." Core commands should be 3-5 max.

## Archived commands:

| Command | Reason | When to restore |
|---------|--------|----------------|
| /deploy-testnet | Rarely used, specific params | Before mainnet prep |
| /audit-old-contracts | One-time use, already done | Never (reference only) |
| /backend-init | Future use | When starting backend |
| /scan-project | Replaced by /catchup + /status | Never |
| /sync-repos | Occasional use | Before major milestones |
